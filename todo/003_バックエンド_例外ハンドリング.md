# 003 バックエンド - 例外ハンドリング

## 概要
API エラー時に統一フォーマットのレスポンスを返す例外ハンドラーを実装する。

## Todo

### エラーレスポンス DTO（`dto/ErrorResponse.java`）
- [ ] `status`（int）、`error`（String）、`message`（String）、`timestamp`（LocalDateTime）フィールドを定義する
- [ ] バリデーションエラー用に `fieldErrors`（List）フィールドを定義する
- [ ] `FieldError` の内部クラス/レコードとして `field`（String）、`message`（String）を定義する

### カスタム例外クラス（`exception/`）
- [ ] `ResourceNotFoundException.java`：リソース未存在・論理削除済みの場合にスローする（404用）
- [ ] `BusinessException.java`：業務ロジック違反（回答3件超など）の場合にスローする（400用）

### グローバル例外ハンドラー（`exception/GlobalExceptionHandler.java`）
- [ ] `@ControllerAdvice` を付与する
- [ ] `ResourceNotFoundException` → 404 Not Found を返す
- [ ] `BusinessException` → 400 Bad Request を返す
- [ ] `MethodArgumentNotValidException`（Bean Validation エラー）→ 400 + `fieldErrors` を返す
- [ ] `Exception`（予期しない例外）→ 500 Internal Server Error を返す（スタックトレースはレスポンスに含めない）
- [ ] エラー発生時に ERROR レベルでログを出力する（500の場合はスタックトレースをログに記録する）

### エラーレスポンスフォーマット確認
- [ ] 以下の形式で返却されることを確認する
  ```json
  {
    "status": 400,
    "error": "Validation Failed",
    "message": "入力内容に誤りがあります。",
    "timestamp": "2026-03-01T10:00:00",
    "fieldErrors": [
      { "field": "title", "message": "タイトルは必須です。" }
    ]
  }
  ```
