# 003 バックエンド - 例外ハンドリング

## 概要
API エラー時に統一フォーマットのレスポンスを返す例外ハンドラーを実装する。

## Todo

### エラーレスポンス DTO（`dto/ErrorResponse.java`）
- [⚪︎] `status`（int）、`error`（String）、`message`（String）、`timestamp`（**OffsetDateTime**）フィールドを定義する
- [⚪︎] `fieldErrors`（`List<FieldError>`）フィールドを定義する。**null ではなく空リストをデフォルト値とする**
- [⚪︎] `FieldError` を内部クラス/レコードとして定義する（`field`（String）、`message`（String））

### カスタム例外クラス（`exception/`）
- [⚪︎] `ResourceNotFoundException.java`：リソース未存在・論理削除済みの場合にスローする（404用）
  - Service 層での使用例：`repository.findById(id).orElseThrow(() -> new ResourceNotFoundException("Resource not found"))`
- [⚪︎] `BusinessException.java`：業務ロジック違反（回答3件超など）の場合にスローする（400用）

### グローバル例外ハンドラー（`exception/GlobalExceptionHandler.java`）
- [⚪︎] `@ControllerAdvice` を付与する
- [⚪︎] `ResourceNotFoundException` → 404 Not Found を返す（WARN ログ）
- [⚪︎] `BusinessException` → 400 Bad Request を返す（WARN ログ）
- [⚪︎] `MethodArgumentNotValidException`（Bean Validation エラー）→ 400 + `fieldErrors` を返す（WARN ログ）
  - `error = "Validation Failed"`、`message = "入力内容に誤りがあります。"` で固定
  - 詳細は `fieldErrors` に格納する
- [⚪︎] `Exception`（予期しない例外）→ 500 Internal Server Error を返す（ERROR ログ＋スタックトレースをログに記録する）
- [⚪︎] レスポンスにスタックトレースを含めない

ログレベル一覧：

| 例外 | ログレベル |
|------|-----------|
| `ResourceNotFoundException` | WARN |
| `BusinessException` | WARN |
| `MethodArgumentNotValidException` | WARN |
| `Exception`（予期しない例外） | ERROR（スタックトレース付き） |

### エラーレスポンスフォーマット確認
- [ ] 以下の形式で返却されることを確認する
  ```json
  {
    "status": 400,
    "error": "Validation Failed",
    "message": "入力内容に誤りがあります。",
    "timestamp": "2026-03-01T10:00:00+09:00",
    "fieldErrors": [
      { "field": "title", "message": "タイトルは必須です。" }
    ]
  }
  ```
- [ ] バリデーションエラー以外では `fieldErrors` が空配列（`[]`）で返ることを確認する
