# 007 フロントエンド - 基盤構築

## 概要
ルーティング設定、型定義、API通信関数を実装し、フロントエンドの基盤を整備する。

## Todo

### パッケージインストール
- [ ] `react-router-dom` v6 がインストールされていることを確認する（なければ `npm install react-router-dom`）

### 型定義（`src/types/index.ts`）
- [ ] `Question` 型を定義する
  ```ts
  type Status = 'UNANSWERED' | 'ANSWERED';
  interface Question {
    id: number;
    title: string;
    content: string;
    questioner: string;
    status: Status;
    createdAt: string;
    updatedAt: string;
    answers: Answer[];
  }
  ```
- [ ] `Answer` 型を定義する
  ```ts
  interface Answer {
    id: number;
    questionId: number;
    content: string;
    responder: string;
    createdAt: string;
    updatedAt: string;
  }
  ```
- [ ] `QuestionRequest` 型を定義する（`title`, `content`, `questioner`）
- [ ] `AnswerRequest` 型を定義する（`content`, `responder`）
- [ ] `ErrorResponse` 型を定義する（`status`, `error`, `message`, `timestamp`, `fieldErrors?`）
- [ ] `FieldError` 型を定義する（`field`, `message`）

### API通信関数（`src/api/questions.ts`）
- [ ] `getQuestions(params: { keyword?, status?, sort? })` → `Question[]`
- [ ] `getQuestion(id: number)` → `Question`
- [ ] `createQuestion(data: QuestionRequest)` → `Question`
- [ ] `updateQuestion(id: number, data: QuestionRequest)` → `Question`
- [ ] `deleteQuestion(id: number)` → `{ message: string }`

### API通信関数（`src/api/answers.ts`）
- [ ] `createAnswer(questionId: number, data: AnswerRequest)` → `Answer`
- [ ] `updateAnswer(answerId: number, data: AnswerRequest)` → `Answer`
- [ ] `deleteAnswer(answerId: number)` → `{ message: string }`

### API通信の共通処理
- [ ] ベースURL `http://localhost:8080` を定数または環境変数で管理する
- [ ] API エラー時（4xx / 5xx）に `ErrorResponse` をパースしてスローする処理を実装する

### ルーティング（`src/App.tsx`）
- [ ] `BrowserRouter` でアプリをラップする
- [ ] 以下のルートを定義する
  - `/` → `TopPage`
  - `/questions/new` → `QuestionNewPage`
  - `/questions/unanswered` → `UnansweredListPage`
  - `/questions/:id` → `QuestionDetailPage`
  - `/questions/:id/edit` → `QuestionEditPage`
  - `/qa` → `QaListPage`

- [ ] TypeScript のコンパイルエラーがないことを確認する（`npm run build`）
