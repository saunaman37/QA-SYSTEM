# 001 環境構築

## 概要
バックエンド・データベースの起動環境を整備し、開発に必要な設定を行う。

## Todo

### Docker / MySQL（`backend/docker-compose.yml`）
- [⚪︎] `backend/docker-compose.yml` で MySQL 8.0 コンテナが起動することを確認する
- [⚪︎] 以下の設定が含まれていることを確認する
  - charset: `utf8mb4`
  - collation: `utf8mb4_0900_ai_ci`
  - timezone: `Asia/Tokyo`
- [⚪︎] `qa_mysql_data` ボリュームでデータが永続化されることを確認する
- [⚪︎] `environment` セクションに DB 接続情報（`SPRING_DATASOURCE_URL` 等）を設定する

### 環境変数
- [⚪︎] `backend/.env` ファイルを作成し、以下の変数を定義する（docker-compose 用）
  ```
  SPRING_DATASOURCE_URL=jdbc:mysql://localhost:3306/qadb?serverTimezone=Asia/Tokyo&characterEncoding=UTF-8
  SPRING_DATASOURCE_USERNAME=qauser
  SPRING_DATASOURCE_PASSWORD=qapassword
  ```
- [⚪︎] `backend/.env` を `.gitignore` に追加する

### バックエンド - パッケージ構成
- [⚪︎] ベースパッケージ `com.example.qabackend` を確認する（アンダースコアなし）
- [⚪︎] 以下のパッケージディレクトリを作成する
  - `controller/`
  - `service/`
  - `repository/`
  - `entity/`
  - `dto/`
  - `exception/`

### バックエンド - application.yml
- [⚪︎] DB接続情報を環境変数参照形式に設定する
  ```yaml
  spring:
    datasource:
      url: ${SPRING_DATASOURCE_URL}
      username: ${SPRING_DATASOURCE_USERNAME}
      password: ${SPRING_DATASOURCE_PASSWORD}
  ```
- [⚪︎] `server.error.include-stacktrace: never` を設定する
- [⚪︎] JVM タイムゾーンを `Asia/Tokyo` に設定する
  ```yaml
  spring:
    jpa:
      properties:
        hibernate:
          jdbc:
            time_zone: Asia/Tokyo
  ```

### CORS設定
- [⚪︎] `WebMvcConfigurer` を実装したグローバル CORS 設定クラスを作成する
- [⚪︎] `/api/**` に対して `http://localhost:5173` からのリクエストを許可する（GET / POST / PUT / DELETE）

### 動作確認
- [⚪︎] `docker compose -f backend/docker-compose.yml up -d` で MySQL が起動することを確認する
- [⚪︎] `cd backend && ./mvnw spring-boot:run` でバックエンドが起動することを確認する
- [⚪︎] `GET http://localhost:8080/health` が 200 を返すことを確認する
