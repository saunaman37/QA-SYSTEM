# 001 環境構築

## 概要
バックエンド・データベースの起動環境を整備し、開発に必要な設定を行う。

## Todo

### Docker / MySQL
- [ ] `docker-compose.yml` で MySQL 8.0 コンテナが起動することを確認する
- [ ] charset: utf8mb4、timezone: Asia/Tokyo の設定を確認する
- [ ] `qa_mysql_data` ボリュームでデータが永続化されることを確認する

### バックエンド - パッケージ構成
- [ ] パッケージを `com.example.qabackend` に統一する（アンダースコアなし）
- [ ] `controller/` `service/` `repository/` `entity/` `dto/` `exception/` の各パッケージディレクトリを作成する

### バックエンド - application.yml
- [ ] DB接続情報を環境変数参照形式に変更する
  ```yaml
  spring:
    datasource:
      url: ${SPRING_DATASOURCE_URL}
      username: ${SPRING_DATASOURCE_USERNAME}
      password: ${SPRING_DATASOURCE_PASSWORD}
  ```
- [ ] `server.error.include-stacktrace=never` を設定する
- [ ] タイムゾーン設定（Asia/Tokyo）を追加する

### 環境変数
- [ ] ローカル用の `.env` ファイルを作成する（`SPRING_DATASOURCE_URL` 等を記載）
- [ ] `.env` を `.gitignore` に追加する
- [ ] `docker-compose.yml` の `environment` セクションに DB接続情報を設定する

### CORS設定
- [ ] Spring Boot に CORS 設定を追加し、`http://localhost:5173` を許可する

### 動作確認
- [ ] `docker compose -f backend/docker-compose.yml up -d` でMySQLが起動することを確認する
- [ ] `./mvnw spring-boot:run` でバックエンドが起動することを確認する
- [ ] `GET /health` が 200 を返すことを確認する
