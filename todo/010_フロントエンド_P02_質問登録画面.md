### API連携
- [x] 「登録」ボタン押下時の順序：
  1. 既存のフィールドエラー・APIエラーをクリアする
  2. フロントバリデーションを実行する
  3. バリデーションエラーがある場合は API を呼ばない
  4. バリデーションエラーが無い場合のみ `POST /api/questions` を呼ぶ
  5. 成功時：`/qa` へ遷移する
  6. 失敗時：同画面に留まり、入力内容を保持したままエラーを表示する
- [x] 400 かつ `fieldErrors` あり：`fieldErrors` を各フィールドに紐づけて表示する
- [x] 500 / 通信失敗 / 400 だが `fieldErrors` なし / 想定外レスポンス：「システムエラーが発生しました。しばらくしてから再度お試しください。」をフォーム上部に表示する
- [x] API通信中は「登録」ボタンを非活性にして二重送信を防止する。通信完了後に再度活性化する
