# 004 バックエンド - 質問API

## 概要
質問に関する REST API（一覧取得・詳細取得・登録・更新・削除）を実装する。

## Todo

### DTO

- [ ] `QuestionRequest.java`：`title`・`content`・`questioner` フィールド（`status` は含めない）
  - [ ] Bean Validation アノテーションを付与する（`@NotBlank`, `@Size`）
- [ ] `QuestionResponse.java`：`id`・`title`・`content`・`questioner`・`status`・`createdAt`・`updatedAt` フィールド
- [ ] `QuestionDetailResponse.java`：`QuestionResponse` + `answers`（`AnswerResponse` のリスト）

### リポジトリ（`QuestionRepository.java`）
- [ ] `findByDeletedAtIsNull(Sort sort)` を定義する（全件取得）
- [ ] キーワード・ステータス絞り込み対応のカスタムクエリメソッドを定義する（`@Query` または Specification）

### サービス（`QuestionService.java`）
- [ ] `@Service` を付与する
- [ ] `getQuestions(String keyword, String status, String sort)`：一覧取得（`deleted_at IS NULL` 条件必須）
  - キーワードがある場合：`title` / `content` / 紐づく回答の `content` を LIKE 検索
  - `status` がある場合：ステータスで絞り込む
  - `sort` に応じて `created_at` の昇順/降順で返す
- [ ] `getQuestion(Long id)`：詳細取得（`deleted_at IS NULL`、紐づく有効回答も含む）、存在しない場合は `ResourceNotFoundException`
- [ ] `createQuestion(QuestionRequest req)`：登録（`status` は `UNANSWERED` 固定）
- [ ] `updateQuestion(Long id, QuestionRequest req)`：更新（論理削除済みは `ResourceNotFoundException`、`status` は変更しない）
- [ ] `deleteQuestion(Long id)`：論理削除（後述の業務ロジックチケット参照）

### コントローラー（`QuestionController.java`）
- [ ] `@RestController` `@RequestMapping("/api/questions")` を付与する
- [ ] `GET /api/questions`：クエリパラメータ `keyword`・`status`・`sort` を受け取り一覧を返す（200）
- [ ] `GET /api/questions/{id}`：詳細を返す（200）、存在しない場合は 404
- [ ] `POST /api/questions`：登録（201）、バリデーションエラー時は 400
- [ ] `PUT /api/questions/{id}`：更新（200）、存在しない場合は 404
- [ ] `DELETE /api/questions/{id}`：論理削除（200 + `{"message": "質問を削除しました。"}`）、存在しない場合は 404

### ログ出力
- [ ] 質問登録成功時：`[INFO] [QUESTION_CREATE] id=? title=?`
- [ ] 質問更新成功時：`[INFO] [QUESTION_UPDATE] id=?`
- [ ] 質問削除成功時：`[INFO] [QUESTION_DELETE] id=?`
