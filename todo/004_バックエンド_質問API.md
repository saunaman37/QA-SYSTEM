# 004 バックエンド - 質問API

## 概要
質問に関する REST API（一覧取得・詳細取得・登録・更新・削除）を実装する。

## Todo

### DTO

- [⚪︎] `QuestionRequest.java`：`title`・`content`・`questioner` フィールド（`status` は含めない）
  - [⚪︎] Bean Validation アノテーションを付与する
    - `title`：`@NotBlank` + `@Size(max = 100)`
    - `content`：`@NotBlank` + `@Size(max = 1000)`
    - `questioner`：`@NotBlank` + `@Size(max = 50)`
- [⚪︎] `QuestionResponse.java`：`id`・`title`・`content`・`questioner`・`status`・`createdAt`・`updatedAt` フィールド
  - 一覧 API（`GET /api/questions`）のレスポンス要素として使用する（`List<QuestionResponse>`）
- [⚪︎] `QuestionDetailResponse.java`：`QuestionResponse` の全フィールド + `answers`（`List<AnswerResponse>`）
  - 詳細 API（`GET /api/questions/{id}`）のレスポンスとして使用する
  - `answers` は `deleted_at IS NULL` の回答のみ、`created_at ASC` 順で格納する

### リポジトリ（`QuestionRepository.java`）
- [⚪︎] `findByDeletedAtIsNull(Sort sort)` を定義する（全件取得）
- [⚪︎] キーワード・ステータス絞り込み対応のカスタムクエリメソッドを定義する（`@Query` または Specification）

### サービス（`QuestionService.java`）
- [⚪︎] `@Service` `@Transactional` を付与する
- [⚪︎] `getQuestions(String keyword, String status, String sort)`：一覧取得
  - `deleted_at IS NULL` を必ず WHERE 条件に含める
  - `sort` は `asc` / `desc` のみ許可する。未指定時は `desc`。不正値の場合は `BusinessException`（400）を返す
  - `status` は `UNANSWERED` / `ANSWERED` のみ許可する。不正値の場合は `BusinessException`（400）を返す
  - キーワードがある場合：`title` / `content` を LIKE 検索（回答テーブルは JOIN しない）
  - `status` がある場合：ステータスで絞り込む
  - `sort` に応じて `created_at` の昇順/降順で返す
- [⚪︎] `getQuestion(Long id)`：詳細取得
  - `deleted_at IS NULL` 条件で取得する
  - 存在しない・論理削除済みの場合は `ResourceNotFoundException` を返す
  - 回答は `deleted_at IS NULL` かつ `created_at ASC` 順で取得する
- [⚪︎] `createQuestion(QuestionRequest req)`：登録（`status` は `UNANSWERED` 固定）
- [⚪︎] `updateQuestion(Long id, QuestionRequest req)`：更新
  - 論理削除済みの場合は `ResourceNotFoundException` を返す
  - `status` は変更しない（フロントエンドからの `status` 送信は禁止）
- [⚪︎] `deleteQuestion(Long id)`：論理削除
  - 論理削除済みの場合は `ResourceNotFoundException` を返す
  - 紐づく有効回答（`deleted_at IS NULL`）を同一トランザクション内で論理削除する

### コントローラー（`QuestionController.java`）
- [⚪︎] `@RestController` `@RequestMapping("/api/questions")` を付与する
- [⚪︎] `GET /api/questions`：クエリパラメータ `keyword`・`status`・`sort` を受け取り `List<QuestionResponse>` を返す（200）
- [⚪︎] `GET /api/questions/{id}`：`QuestionDetailResponse` を返す（200）、存在しない場合は 404
- [⚪︎] `POST /api/questions`：`@Valid` を付与し登録（201）、バリデーションエラー時は 400
- [⚪︎] `PUT /api/questions/{id}`：`@Valid` を付与し更新（200）、存在しない場合は 404
- [⚪︎] `DELETE /api/questions/{id}`：論理削除（200 + `{"message": "質問を削除しました。"}`）、存在しない場合は 404

### ログ出力
- [⚪︎] 質問登録成功時：`[INFO] [QUESTION_CREATE] id=? title=?`
- [⚪︎] 質問更新成功時：`[INFO] [QUESTION_UPDATE] id=?`
- [⚪︎] 質問削除成功時：`[INFO] [QUESTION_DELETE] id=?`
