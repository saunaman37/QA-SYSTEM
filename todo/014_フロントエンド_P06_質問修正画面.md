# 014 フロントエンド - P-06 質問修正画面

## 概要
既存質問の内容を編集・更新する画面を実装する。

## 画面パス
`/questions/:id/edit` → `QuestionEditPage.tsx`

## Todo

### UI実装（`src/pages/QuestionEditPage.tsx`）

#### 初期表示状態
- [ ] `GET /api/questions/{id}` の完了まで「読み込み中...」を表示する（フォームは表示しない）
- [ ] 取得中は「更新」ボタンを表示しない（または非活性にする）
- [ ] 取得成功時：APIレスポンスの値をフォームに展開し、`fieldErrors` と `apiError` をクリアする
- [ ] `id` パラメータが変わった場合は再取得してフォームを再展開する

#### 404 時
- [ ] 「対象データが見つかりません。」を表示する
- [ ] フォームおよび「更新」ボタンは表示しない
- [ ] 「戻る」ボタンのみ表示する

#### 500 時（初期表示）
- [ ] 「システムエラーが発生しました。しばらくしてから再度お試しください。」を表示する
- [ ] フォームは表示しない
- [ ] 「戻る」ボタンのみ表示する

#### フォーム構成（取得成功時に表示）
- [ ] タイトル入力欄（テキスト、100文字以内）
- [ ] 質問内容入力欄（テキストエリア、500文字以内）
- [ ] 質問者入力欄（テキスト、50文字以内）
- [ ] 「更新」ボタン
- [ ] 「戻る」ボタン → クリックで `/` へ遷移する（`history.back` は使用しない）

### API連携

#### 初期表示
- [ ] マウント時（および `id` 変更時）に `GET /api/questions/{id}` を呼ぶ
- [ ] 200：レスポンスの値をフォームに展開し、フォームを表示する
- [ ] 404：「対象データが見つかりません。」を表示する
- [ ] 500：「システムエラーが発生しました。しばらくしてから再度お試しください。」を表示する

#### 更新
- [ ] 「更新」ボタン押下時：フロントバリデーション通過後に `PUT /api/questions/{id}` を呼ぶ
  - リクエストボディに `status` は含めない
- [ ] 送信中は「更新」ボタンを非活性にし、多重送信を防止する
- [ ] 200：`/questions/:id` へリダイレクトする
- [ ] 400 + `fieldErrors`：各フィールドに対応するエラーを表示する（フォーム入力値は保持する）
- [ ] 500：フォーム上部に「システムエラーが発生しました。しばらくしてから再度お試しください。」を表示する（フォーム入力値は保持する）

### バリデーション
- 必須判定は trim 後の空文字をエラーとする
- [ ] タイトル：必須（「タイトルは必須です。」）・100文字以内（「タイトルは100文字以内で入力してください。」）
- [ ] 質問内容：必須（「質問内容は必須です。」）・500文字以内（「質問内容は500文字以内で入力してください。」）
- [ ] 質問者：必須（「質問者は必須です。」）・50文字以内（「質問者は50文字以内で入力してください。」）
- [ ] エラーは各フィールド直下にインライン表示する
