# 013 フロントエンド - P-05 Q&A一覧画面

## 概要
全Q&Aを一覧表示し、キーワード検索・ステータス絞り込み・ソートができる画面を実装する。

## 画面パス
`/qa` → `QaListPage.tsx`

## Todo

### UI実装（`src/pages/QaListPage.tsx`）
- [⚪︎] キーワード検索欄（テキスト入力）を配置する。初期値は空文字
- [⚪︎] 「検索」ボタンを配置する
- [⚪︎] ステータス絞り込み（全件 / 未回答のみ / 回答済のみ）をプルダウンまたはラジオボタンで配置する。初期値は全件
- [⚪︎] ソート選択（新着順 / 古い順）をプルダウンまたはラジオボタンで配置する。初期値は新着順（desc）
- [⚪︎] 一覧テーブルを実装する（カラム：タイトル・質問者・ステータス・質問日時）
  - [⚪︎] タイトルはリンク表示し、クリックで `/questions/:id` へ遷移する
  - [⚪︎] 質問日時はフロントエンドで `YYYY/MM/DD HH:mm` 形式に整形して表示する
  - [⚪︎] ステータスは `UNANSWERED` → 「未回答」、`ANSWERED` → 「回答済」に変換して表示する（APIの値をそのまま表示しない）
  - [⚪︎] 初回取得前は0件メッセージを表示しない
  - [⚪︎] 取得結果が0件の場合は、テーブルの代わりに「該当するQ&Aはありません。」を表示する
- [⚪︎] 「戻る」ボタンを配置する → `useNavigate` で `/` へ遷移する（`history.back` は使用しない）
- [⚪︎] 画面上部に共通エラー表示領域を設ける
- [⚪︎] 一覧データは `QuestionSummary` 型を使用する（詳細情報・回答一覧は扱わない）

### 検索ルール
- [⚪︎] 検索実行契機は「検索」ボタン押下時のみとする（入力・選択変更時は即時検索しない）
- [⚪︎] キーワードは `trim()` 後に使用する
- [⚪︎] trim 後に空文字の場合は `keyword` パラメータを付与しない
- [⚪︎] ステータスが「全件」の場合は `status` パラメータを付与しない
- [⚪︎] ステータスが「未回答のみ」の場合は `status=UNANSWERED`
- [⚪︎] ステータスが「回答済のみ」の場合は `status=ANSWERED`
- [⚪︎] ソートは `sort=desc`（新着順） / `sort=asc`（古い順）を付与する
- [⚪︎] 検索結果の再フィルタはフロントエンドでは行わない（APIに委ねる）

### API連携
- [⚪︎] 初期表示時：`GET /api/questions?sort=desc` を呼ぶ（`status`・`keyword` は付与しない）
- [⚪︎] 検索ボタン押下時：上記の検索ルールに従いクエリパラメータを組み立てて呼ぶ
- [⚪︎] API通信中は「検索」ボタンを非活性にして多重リクエストを防止する。通信完了後に再度活性化する
- [⚪︎] APIエラー時：画面上部に共通エラーメッセージを表示する。前回一覧を空で上書きしない
