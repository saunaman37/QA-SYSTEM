# 012 フロントエンド - P-04 質問詳細・回答画面

## 概要
質問の詳細を表示し、回答の登録・修正・削除を行う画面を実装する。

## 画面パス
`/questions/:id` → `QuestionDetailPage.tsx`

## Todo

### UI実装（`src/pages/QuestionDetailPage.tsx`）

#### 質問エリア
- [ ] タイトルを表示する
- [ ] 質問内容を表示する
- [ ] 質問者を表示する
- [ ] 質問日時を表示する（`YYYY/MM/DD HH:mm` 形式）
- [ ] ステータスは `GET /api/questions/{id}` のレスポンスの `status` をそのまま使用し、バッジ等で視覚的に区別する（未回答 / 回答済）
- [ ] 「質問を修正」ボタン → `/questions/:id/edit` へ遷移する

- [ ] 各回答に「修正」ボタンを配置する → 対象回答オブジェクト全体を `AnswerEditModal` に渡してモーダルを開く
- [ ] 各回答に「削除」ボタンを配置する → 削除確認ダイアログを開く
- [ ] 「回答を追加」ボタンを配置する → 回答登録モーダルを開く
- [ ] 有効回答数が `answers.length >= 3` の場合、「回答を追加」ボタンを非活性にし、ボタン付近に「回答は最大3件までです。」と表示する（削除済み回答は件数に含めない前提）

#### 共通
- [ ] 「戻る」ボタンを配置する → `useNavigate` で `/` へ遷移する（`history.back` は使用しない）

### 初回取得ルール
- [ ] 画面初期表示時に `GET /api/questions/{id}` を呼ぶ
- [ ] 初回取得中は「読み込み中...」を表示する
- [ ] 初回取得完了までは、質問修正・質問削除・回答追加・回答修正・回答削除の操作を行えないようにする
- [ ] 初回取得で 404 の場合：「対象データが見つかりません。」をページ全体に表示する（他UIは表示しない）
- [ ] 初回取得で 500 の場合：「システムエラーが発生しました。しばらくしてから再度お試しください。」をページ全体に表示する（他UIは表示しない）

### モーダル連携（チケット008参照）
- [ ] 「回答を追加」ボタン押下時に `AnswerFormModal` を開く
- [ ] 回答登録成功後にモーダルを閉じて詳細データを再取得する
- [ ] 「修正」ボタン押下時に `AnswerEditModal` を開く（対象回答オブジェクト全体を渡す）
- [ ] 回答更新成功後にモーダルを閉じて詳細データを再取得する

### 削除処理
- [ ] 「質問を削除」ボタン押下時に `DeleteConfirmDialog` を開く（文言：「この質問を削除しますか？」）
- [ ] 削除確認後に `DELETE /api/questions/{id}` を呼び、成功時に `/` へ遷移する
- [ ] 「回答を削除」ボタン押下時に `DeleteConfirmDialog` を開く（文言：「この回答を削除しますか？」）
- [ ] 削除確認後に `DELETE /api/answers/{answerId}` を呼び、成功時に詳細データを再取得する

### 多重送信防止
- [ ] 質問削除の DELETE 実行中は確認ボタンを非活性にする
- [ ] 回答削除の DELETE 実行中は確認ボタンを非活性にする
- [ ] 回答登録・更新モーダルの送信中は送信ボタンを非活性にする

### 再取得失敗時の挙動
- [ ] 回答登録・更新・削除後の再取得が失敗した場合、直前まで表示していた質問詳細・回答一覧を保持する
- [ ] エラーメッセージはページ上部の共通エラー領域に表示する（ページ全体を404/500表示に切り替えない）
