# 012 フロントエンド - P-04 質問詳細・回答画面

## 概要
質問の詳細を表示し、回答の登録・修正・削除を行う画面を実装する。

## 画面パス
`/questions/:id` → `QuestionDetailPage.tsx`

## Todo

### UI実装（`src/pages/QuestionDetailPage.tsx`）

#### 質問エリア
- [ ] タイトルを表示する
- [ ] 質問内容を表示する
- [ ] 質問者を表示する
- [ ] 質問日時を表示する（`YYYY/MM/DD HH:mm` 形式）
- [ ] ステータスをバッジ等で表示する（未回答 / 回答済で視覚的に区別）
- [ ] 「質問を修正」ボタン → `/questions/:id/edit` へ遷移する
- [ ] 「質問を削除」ボタン → 削除確認ダイアログを開く

#### 回答エリア
- [ ] 回答一覧（最大3件）を表示する（回答内容・回答者・回答日時・更新日時）
- [ ] 各回答に「修正」ボタンを配置する → 回答修正モーダルを開く
- [ ] 各回答に「削除」ボタンを配置する → 削除確認ダイアログを開く
- [ ] 「回答を追加」ボタンを配置する → 回答登録モーダルを開く
- [ ] 有効回答数が 3 件の場合は「回答を追加」ボタンを非表示または非活性にする

#### 共通
- [ ] 「戻る」ボタンを配置する → `/` へ遷移する

### モーダル連携（チケット008参照）

- [ ] 回答登録成功後に詳細データを再取得して画面を更新する

- [ ] 回答更新成功後に詳細データを再取得して画面を更新する

### 削除処理
- [ ] 「質問を削除」ボタン押下時に `DeleteConfirmDialog` を開く
- [ ] 削除確認後に `DELETE /api/questions/{id}` を呼び、成功時に `/qa` へ遷移する
- [ ] 「回答を削除」ボタン押下時に `DeleteConfirmDialog` を開く
- [ ] 削除確認後に `DELETE /api/answers/{answerId}` を呼び、成功時に詳細データを再取得する

### API連携
- [ ] 画面初期表示時に `GET /api/questions/{id}` を呼ぶ
- [ ] 対象が存在しない場合（404）：「対象データが見つかりません。」を表示する
- [ ] APIエラー（500）：「システムエラーが発生しました。しばらくしてから再度お試しください。」を表示する
