# 005 バックエンド - 回答API

## 概要
回答に関する REST API（登録・更新・削除）を実装する。ステータス自動更新・3件制限・論理削除の業務ロジックを含む。

## 業務ルール

### 質問ステータスの自動更新
- `Question.status` の更新責務は `AnswerService` が担う（フロントエンドからは送信しない）
- 回答登録後、有効回答数が 0 → 1 になった場合：`Question.status` を `ANSWERED` に更新する
- 回答削除後、有効回答数が 1 → 0 になった場合：`Question.status` を `UNANSWERED` に戻す
- 「有効回答数」＝ `deleted_at IS NULL` の回答数

### 回答登録制限
- 1質問につき有効回答は最大3件まで
- 有効回答が3件以上の状態で登録しようとした場合は `BusinessException`（400、「回答は最大3件までです。」）

### 親質問の状態チェック
- 回答登録時に以下を確認する（この順序で）
  1. 親質問の存在チェック（存在しない場合は `ResourceNotFoundException`）
  2. 親質問の論理削除チェック（削除済みの場合は `ResourceNotFoundException`）
  3. 有効回答数チェック（3件以上で `BusinessException`）

### 親質問チェックの適用範囲
- 回答の登録・更新・削除のすべてで、親質問が存在し、かつ `deleted_at IS NULL` であることを確認する
- 条件を満たさない場合は `ResourceNotFoundException` を返す

### 回答件数上限チェックの適用範囲
- 回答件数上限チェック（最大3件）は回答新規登録時のみ実施する
- 回答更新時は件数チェックを行わない

### 更新・削除対象の条件
- 回答更新・削除は `deleted_at IS NULL` の回答のみを対象とする
- 対象が存在しない場合は `ResourceNotFoundException` を返す

### トランザクション
- 回答操作（保存・更新・削除）と `Question.status` 更新は同一トランザクション内で実行する
