# 005 バックエンド - 回答API

## 概要
回答に関する REST API（登録・更新・削除）を実装する。ステータス自動更新・3件制限・論理削除の業務ロジックを含む。

### サービス（`AnswerService.java`）
- [ ] `@Service` を付与する
- [ ] `createAnswer(Long questionId, AnswerRequest req)`：
  - `@Transactional` を付与する
  - 対象質問が存在しない（または論理削除済み）場合は `ResourceNotFoundException`（404）
  - 有効回答数が 3 件以上の場合は `BusinessException`（400、「回答は最大3件までです。」）
  - 回答を保存する
  - 有効回答数が 0 → 1 になった場合（保存後）、質問の `status` を `ANSWERED` に更新する
- [ ] `updateAnswer(Long answerId, AnswerRequest req)`：
  - `@Transactional` を付与する
  - 対象回答が存在しない（または論理削除済み）場合は `ResourceNotFoundException`（404）
  - 回答を更新する
- [ ] `deleteAnswer(Long answerId)`：
  - `@Transactional` を付与する
  - 対象回答が存在しない（または論理削除済み）場合は `ResourceNotFoundException`（404）
  - 回答を論理削除する（`deleted_at` に現在時刻をセット）
  - 削除後の有効回答数が 0 になった場合、質問の `status` を `UNANSWERED` に更新する

### コントローラー（`AnswerController.java`）
- [ ] `POST /api/questions/{id}/answers`：回答登録（201）、質問が存在しない場合は 404、3件超は 400
- [ ] `PUT /api/answers/{answerId}`：回答更新（200）、存在しない場合は 404
- [ ] `DELETE /api/answers/{answerId}`：論理削除（200 + `{"message": "回答を削除しました。"}`）、存在しない場合は 404

### ステータス自動更新ロジックの確認
