# 011 フロントエンド - P-03 回答対象一覧画面

## 概要
未回答質問の一覧を表示し、キーワード検索・ソートができる画面を実装する。

## 画面パス
`/questions/unanswered` → `UnansweredListPage.tsx`

### 検索ルール
- [x] 検索実行契機は「検索」ボタン押下時のみとする（キーワード入力中・ソート変更時は即時検索しない）
- [x] 入力キーワードは trim してから送信する
- [x] trim 後に空文字の場合は `keyword` パラメータを付与せずにAPIを呼ぶ
- [x] 検索結果の再フィルタはフロントエンドでは行わない（APIに委ねる）

### API連携
- [x] 初期表示時：`GET /api/questions?status=UNANSWERED&sort=desc` を呼ぶ（`status=UNANSWERED` は常に固定）
- [x] 検索ボタン押下時：
  - キーワードあり：`GET /api/questions?status=UNANSWERED&keyword=...&sort=...`
  - キーワードなし：`GET /api/questions?status=UNANSWERED&sort=...`
- [x] API通信中は「検索」ボタンを非活性にして多重リクエストを防止する。通信完了後に再度活性化する
- [x] APIエラー時：「データの取得に失敗しました。しばらくしてから再度お試しください。」を共通エラー領域に表示する
- [x] エラー発生時は前回取得済み一覧を保持してよい。ただし初回取得失敗時は一覧を表示しない
